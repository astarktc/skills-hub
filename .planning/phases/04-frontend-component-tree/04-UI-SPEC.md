# Phase 4 -- UI Design Contract

> Visual and interaction contract for the Projects tab UI. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property          | Value                                                      |
| ----------------- | ---------------------------------------------------------- |
| Tool              | none                                                       |
| Preset            | not applicable                                             |
| Component library | none (custom CSS + Tailwind 4 utilities)                   |
| Icon library      | lucide-react 0.562.x                                       |
| Font              | Fira Sans (UI) / Fira Code (mono), loaded via Google Fonts |

**Note:** This project uses hand-rolled CSS variables in `src/index.css` with semantic class names in `src/App.css`. No shadcn, no Radix, no headless UI library. All new Phase 4 components follow the same pattern: semantic CSS classes in `App.css`, CSS variables for theming, Tailwind utilities only for ad-hoc layout.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage                                                          |
| ----- | ----- | -------------------------------------------------------------- |
| xs    | 4px   | Icon gaps (nav-tab gap: 6px rounds to nearest), inline padding |
| sm    | 8px   | Compact element spacing, checkbox gaps                         |
| md    | 16px  | Default element spacing, card internal padding, list gaps      |
| lg    | 24px  | Section padding (header: 24px horizontal), panel padding       |
| xl    | 32px  | Content area padding (skills-list: 32px), page gutters         |
| 2xl   | 48px  | Skill icon size (48px), major section breaks                   |
| 3xl   | 64px  | Not used in this phase                                         |

Exceptions:

- Left panel width: 250px (D-01 locked decision, not a spacing token)
- Header height: 56px (existing, reused)
- Nav tab horizontal padding: 14px (existing pattern, inherited from Header.tsx)
- Checkbox input size: 18px (existing pattern from pick-item-checkbox)
- Status dot: 8px diameter (aggregate sync indicator in project list)

## Interaction Contracts

### Project Selection (left panel)

| Trigger            | Visual Response                                                | Backend Call                                    | State Change                |
| ------------------ | -------------------------------------------------------------- | ----------------------------------------------- | --------------------------- |
| Click project item | Item gets `selected` class (accent-soft-bg), right panel loads | `list_project_tools`, `get_project_assignments` | `selectedProjectId` updates |
| Hover project item | Background shifts to `--bg-element-hover`                      | None                                            | None                        |

### Matrix Checkbox Toggle (D-06: wait-for-backend)

| Step                          | Visual                                                                           | Duration   |
| ----------------------------- | -------------------------------------------------------------------------------- | ---------- |
| 1. User clicks unchecked cell | Cell background goes `pending` (gray tint), checkbox shows indeterminate/spinner | Immediate  |
| 2. Backend responds success   | Cell background transitions to `synced` (green tint), checkbox shows checked     | ~100-500ms |
| 3. Backend responds error     | Cell background transitions to `error` (red tint), checkbox reverts to unchecked | ~100-500ms |

Uncheck flow (unassign):
| Step | Visual | Duration |
|------|--------|----------|
| 1. User clicks checked cell | Cell background goes `pending`, checkbox shows spinner | Immediate |
| 2. Backend responds success | Cell background clears to default, checkbox shows unchecked | ~100-500ms |
| 3. Backend responds error | Cell background transitions to `error`, checkbox stays checked | ~100-500ms |

### Bulk Assign (D-07)

| Trigger                               | Visual Response                                                      |
| ------------------------------------- | -------------------------------------------------------------------- |
| Click "All Tools" button on skill row | All unchecked cells in that row go to `pending` state simultaneously |
| Backend responds per-tool             | Each cell updates individually from `pending` to `synced` or `error` |

### Error Cell (D-09)

| Trigger               | Visual Response                                                    |
| --------------------- | ------------------------------------------------------------------ |
| Hover red-tinted cell | Tooltip shows error message prefixed with "Sync failed: "          |
| Click red-tinted cell | Retries sync -- cell goes to `pending`, follows normal toggle flow |

### Add Project Flow (D-10, D-11)

| Step                         | UI Element                                                    | Behavior                                                             |
| ---------------------------- | ------------------------------------------------------------- | -------------------------------------------------------------------- |
| 1. Click "Add Project"       | Opens AddProjectModal                                         | Modal with folder picker + manual path input                         |
| 2. Enter/pick path           | Inline duplicate validation fires                             | If duplicate, show inline error below input, disable Register button |
| 3. Click "Register"          | Modal closes, project appears in list selected                | Backend: `register_project`                                          |
| 4. Auto-open ToolConfigModal | Shows tool checkbox list with auto-detected tools pre-checked | Backend: tool detection (existing)                                   |
| 5. Confirm tools             | Tools saved, matrix columns appear                            | Backend: `add_project_tool` per selected tool                        |

### Remove Project Flow (D-15)

| Step                                            | UI Element                                                | Behavior                    |
| ----------------------------------------------- | --------------------------------------------------------- | --------------------------- |
| 1. Right-click or action button on project item | Context action triggers RemoveProjectModal                | Matches DeleteModal pattern |
| 2. Confirm removal                              | Modal closes, project removed from list, selection clears | Backend: `remove_project`   |

### Loading States (D-21)

| Area                                 | Loading Pattern                                                |
| ------------------------------------ | -------------------------------------------------------------- |
| Project list (initial mount)         | 3 skeleton rows (shimmer), each 48px tall with rounded corners |
| Assignment matrix (project selected) | Grid skeleton: 4 rows x 3 columns of shimmer rectangles        |
| Individual cell (toggle in progress) | Cell-level spinner replacing checkbox icon                     |

Skeleton implementation: CSS `@keyframes shimmer` (already exists in App.css as `.progress-fill` animation) applied to placeholder `<div>` elements with `--bg-element` background and animated gradient overlay.

| ------------------ | ---------------------------------------------------------------------------- |

| Loading skeletons  | `aria-hidden="true"` on skeleton elements, `aria-busy="true"` on container   |
