1. Modify `list_assignments_with_staleness` in `src-tauri/src/core/project_sync.rs` (lines 224-260):

The current logic only checks staleness for copy-mode synced targets. It needs to be expanded to also detect missing source and missing target for ALL assignments, and critically, to recalculate status when both source and target exist REGARDLESS of current DB status (so that "missing" assignments auto-recover per D-07).

Replace the function body with this early-return logic (keeping the same signature):

```rust
pub fn list_assignments_with_staleness(
    store: &SkillStore,
    project_id: &str,
) -> Result<Vec<ProjectSkillAssignmentRecord>> {
    let assignments = store.list_project_skill_assignments(project_id)?;
    let mut result = Vec::with_capacity(assignments.len());

    for mut assignment in assignments {
        // --- Resolve source and target existence ---
        let skill_opt = store.get_skill_by_id(&assignment.skill_id).ok().flatten();
        let source_exists = skill_opt
            .as_ref()
            .map(|s| Path::new(&s.central_path).exists())
            .unwrap_or(false);

        let target_exists = if let Some(ref skill) = skill_opt {
            if let Some(adapter) = tool_adapters::adapter_by_key(&assignment.tool) {
                let project = store.get_project_by_id(&assignment.project_id).ok().flatten();
                project.map(|p| {
                    let target = resolve_project_sync_target(
                        Path::new(&p.path),
                        adapter.relative_skills_dir,
                        &skill.name,
                    );
                    target.exists() || target.symlink_metadata().is_ok()
                }).unwrap_or(false)
            } else {
                false
            }
        } else {
            false
        };

        // --- D-04/D-05: Early-return missing when source or target absent ---
        // These are early-continues: if source is gone, nothing else matters.
        // If target is gone (for assignments that were previously deployed), mark missing.
        if !source_exists {
            // D-04: Source absent -> missing, persist, continue to next
            if assignment.status != "missing" {
                let _ = store.update_assignment_status(
                    &assignment.id, "missing", None, None, None, None,
                );
            }
            assignment.status = "missing".to_string();
            result.push(assignment);
            continue;
        }

        if !target_exists
            && (assignment.status == "synced"
                || assignment.status == "stale"
                || assignment.status == "missing")
        {
            // D-05: Target absent for a previously-deployed assignment -> missing
            // Note: includes status=="missing" so a missing assignment whose target
            // is still gone stays missing even though source came back.
            if assignment.status != "missing" {
                let _ = store.update_assignment_status(
                    &assignment.id, "missing", None, None, None, None,
                );
            }
            assignment.status = "missing".to_string();
            result.push(assignment);
            continue;
        }

        // --- D-07: Both source and target exist -> recalculate staleness ---
        // This block runs for ANY current DB status (including "missing").
        // When a previously-missing assignment has both source and target restored,
        // it falls through here and gets recalculated to "synced" or "stale".
        if source_exists && target_exists {
            if assignment.mode == "copy" {
                // Copy mode: compare content hashes to detect staleness
                if let Some(ref skill) = skill_opt {
                    let source = Path::new(&skill.central_path);
                    if let Ok(current_hash) = content_hash::hash_dir(source) {
                        let is_stale = assignment.content_hash.as_deref() != Some(&current_hash);
                        let new_status = if is_stale { "stale" } else { "synced" };
                        if assignment.status != new_status {
                            let _ = store.update_assignment_status(
                                &assignment.id,
                                new_status,
                                None,
                                None,
                                None,
                                if !is_stale { Some(&current_hash) } else { None },
                            );
                        }
                        assignment.status = new_status.to_string();
                    }
                    // If hash_dir fails, leave status unchanged
                }
            } else {
                // Symlink mode: if both exist, it is synced
                // (symlinks point to source, so existence of both = working link)
                if assignment.status == "missing" {
                    let _ = store.update_assignment_status(
                        &assignment.id, "synced", None, None, None, None,
                    );
                    assignment.status = "synced".to_string();
                }
            }
        }

        result.push(assignment);
    }

    Ok(result)
}
```

Key design choices and why they satisfy D-07:

- **Early-continue pattern:** Source-absent and target-absent are early `continue` branches. If either is missing, we mark "missing" and move on. This is clean and unambiguous.
- **Both-exist block runs for ANY DB status:** The critical fix. The `if source_exists && target_exists` block is NOT gated on `assignment.status == "synced"`. An assignment with DB status "missing" where source and target have been restored will enter this block and get recalculated to "synced" or "stale". This directly satisfies D-07.
- **Target-absent check includes status "missing":** A previously-missing assignment where source reappears but target is still gone stays "missing" (correct -- the target hasn't been re-synced yet).
- **Copy mode recalculation:** Compares current source hash against stored hash. If different -> "stale". If same -> "synced". Persists the new status.
- **Symlink mode recovery:** If both source and target exist (symlink intact), and status was "missing", update to "synced".
