2. In `update_managed_skill_from_source`, after the existing tool-target re-sync loop (which ends at line 984, just before `Ok(UpdateResult {...})`), insert a new block that re-syncs copy-mode project assignments:

   ```rust
   // Re-sync copy-mode project skill assignments so project copies stay current.
   // Symlinks auto-update since they point to the central path that was just refreshed.
   let project_assignments = store.list_project_skill_assignments_by_skill(skill_id)?;
   for pa in project_assignments {
       let force_copy = pa.mode == "copy" || pa.tool == "cursor";
       if !force_copy {
           continue;
       }
       let project = match store.get_project_by_id(&pa.project_id)? {
           Some(p) => p,
           None => continue,
       };
       let project_path = PathBuf::from(&project.path);
       if !project_path.exists() {
           continue;
       }
       let adapter = match adapter_by_key(&pa.tool) {
           Some(a) => a,
           None => continue,
       };
       let target = resolve_project_sync_target(
           &project_path,
           adapter.relative_skills_dir,
           &record.name,
       );
       match sync_dir_copy_with_overwrite(&central_path, &target, true) {
           Ok(_outcome) => {
               let _ = store.update_assignment_status(
                   &pa.id,
                   "synced",
                   None,
                   Some(now),
                   Some("copy"),
                   content_hash.as_deref(),
               );
               updated_targets.push(format!("project:{}:{}", pa.project_id, pa.tool));
           }
           Err(e) => {
               log::warn!("failed to re-sync project assignment {}: {:#}", pa.id, e);
               let _ = store.update_assignment_status(
                   &pa.id,
                   "error",
                   Some(&format!("{:#}", e)),
                   None,
                   None,
                   None,
               );
           }
       }
   }
   ```

Key design decisions per CONTEXT.md:

- Copy-mode only (symlinks auto-update) -- includes Cursor force-copy guard matching line 967 pattern.
- Inline after existing tool-target loop -- keeps all update propagation in one function.
- Updates content_hash on assignment records after re-copying.
- Reuses `updated_targets` Vec with `project:` prefix (Claude's discretion area -- avoids changing UpdateResult/DTO/frontend types).
- Gracefully skips: orphaned assignments (no project), missing project dirs, unknown tool adapters.
- Error handling follows project*sync.rs pattern: `log::warn!` + `let * = store.update_assignment_status(...)` for failures, no abort.
