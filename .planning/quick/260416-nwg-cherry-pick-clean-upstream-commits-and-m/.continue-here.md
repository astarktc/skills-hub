## Addendum: Phase C Deep Dive -- Commit 1826e2a Conflict Resolution

### What upstream's commit does (the 6 changes)

**1. New helper functions (no conflict -- purely additive):**

- `normalize_github_skill_subpath()` -- Strips trailing `/SKILL.md` from GitHub URLs (so `user/repo/tree/main/skills/foo/SKILL.md` -> `skills/foo`). Also handles `subpath == "SKILL.md"` -> `"."`. Case-insensitive for the suffix strip.
- `ensure_installable_skill_dir()` -- Validates a directory is a real skill dir before installing. Bails with `SKILL_INVALID|missing_skill_md|...` if not. Called at 4 insertion points in `install_git_skill` and `install_git_skill_from_selection`.
- `is_hidden_dir_name()`, `is_known_root_scan_dir()`, `is_skill_container_dir_name()` -- Small predicates used by `collect_skill_dirs`.

**2. Refactored `collect_skill_dirs()` replacing both `scan_skill_candidates_in_dir()` AND `count_skills_in_repo()` (CONFLICT ZONE):**

Upstream replaces the inline scanning logic with a unified `collect_skill_dirs()` that returns `Vec<PathBuf>`. Then both `scan_skill_candidates_in_dir` and `count_skills_in_repo` become thin wrappers:

```rust
// upstream's new version
fn collect_skill_dirs(repo_dir: &Path) -> Vec<PathBuf> {
    let mut out = Vec::new();
    // 1) Fast path: known skill locations such as skills/* and .claude/skills/*.
    for base in SKILL_SCAN_BASES {
        push_skill_dirs_from_base(&mut out, &repo_dir.join(base));
    }
    // 2) Root-level skills + 3) container dirs (dirs with "skill" in name)
    if let Ok(rd) = std::fs::read_dir(repo_dir) {
        for entry in rd.flatten() {
            // skip hidden, skip known bases
            if is_skill_dir(&p) { out.push(p); }
            else if is_skill_container_dir_name(&dir_name) {
                push_skill_dirs_from_base(&mut out, &p);  // one level deep
            }
        }
    }
    out.sort(); out.dedup(); out
}
```

**3. Simplified `list_git_skills()` (CONFLICT ZONE):**

The ~67-line inline scanning loop gets replaced with `for p in collect_skill_dirs(&repo_dir) { ... }` -- about 12 lines. Also adds a new branch: when a specific subpath exists but `is_skill_dir` is false, it runs `collect_skill_dirs` on that subpath (treating it as a container).

**4. `ensure_installable_skill_dir` guard inserted at 4 copy sites** -- before `copy_dir_recursive` in `install_git_skill` (sparse path, clone-subpath path, clone-root path) and in `install_git_skill_from_selection`.

**5. `normalize_github_skill_subpath` in `parse_github_url`** -- The URL parser now strips `/SKILL.md` from parsed subpaths.

**6. `is_skill_dir` stays case-SENSITIVE** -- upstream's version is `p.join("SKILL.md").exists()`. **This is a regression we must fix.**
