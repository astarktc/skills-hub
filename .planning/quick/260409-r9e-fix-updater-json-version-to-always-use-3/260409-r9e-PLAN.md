<objective>
Ensure the updater.json always contains a valid 3-part semver version (x.y.z), even when
the git tag uses a shortened form like `v1.0`. Also add validation to `version.mjs set`
so it rejects non-3-part versions at the source.

Purpose: Tauri's updater requires strict 3-part semver. A tag like `v1.0` produces `1.0`
in updater.json, which breaks update detection for clients.

Output: Two hardened files that prevent 2-part versions from reaching updater.json.
</objective>

<context>
@.github/workflows/release.yml (line ~464 — VERSION="${TAG#v}" lacks normalization)
@scripts/version.mjs (set command accepts any string without semver validation)
</context>

<tasks>

<task type="auto">
  <name>Task 1: Normalize VERSION in release workflow to 3-part semver</name>
  <files>.github/workflows/release.yml</files>
  <action>
In the "Generate updater.json" step (~line 464), after `VERSION="${TAG#v}"`, add a
normalization block that pads VERSION to 3 parts:

```bash
# Normalize to 3-part semver (e.g., "1.0" -> "1.0.0", "1" -> "1.0.0")
IFS='.' read -ra parts <<< "$VERSION"
while [ ${#parts[@]} -lt 3 ]; do
  parts+=("0")
done
VERSION="${parts[0]}.${parts[1]}.${parts[2]}"
```

Insert this immediately after line 464 (`VERSION="${TAG#v}"`), before `PUB_DATE=...`.
No other changes to the file.
</action>
<verify>
grep -A8 'VERSION="\${TAG#v}"' .github/workflows/release.yml | grep -q 'Normalize to 3-part semver'
</verify>
<done>The "Generate updater.json" step normalizes VERSION to x.y.z before writing the JSON, so tags like v1.0 produce "1.0.0" in updater.json.</done>
</task>

<task type="auto">
  <name>Task 2: Add 3-part semver validation to version.mjs set command</name>
  <files>scripts/version.mjs</files>
  <action>
In the `set` command handler (line ~112), after the `if (!arg)` guard and before
`setPackageJsonVersion(arg)`, add a validation check:

```js
const semverRe = /^\d+\.\d+\.\d+$/;
if (!semverRe.test(arg)) {
  console.error(`Error: Version must be 3-part semver (x.y.z), got "${arg}"`);
  process.exit(1);
}
```

This prevents someone from running `node scripts/version.mjs set 1.0` and having that
2-part version propagate to package.json and tauri.conf.json.
</action>
<verify>
node scripts/version.mjs set 1.0 2>&1 | grep -q '3-part semver' && node scripts/version.mjs set 1.0.0 2>&1 | grep -q 'Version set to 1.0.0'
</verify>
<done>"version.mjs set 1.0" exits with error. "version.mjs set 1.0.0" succeeds. Only x.y.z format accepted.</done>
</task>
