7. Construct full `source_url`: `https://github.com/{owner}/{repo}/tree/{branch}/{path}`

  <action>
Add a post-scrape enrichment phase to `fetch-featured-skills-v2.mjs`. Port the following
functions from `fetch-featured-skills.mjs` (v1) into the v2 script:

1. **`fetchJson(url, retries=3)`** -- Port from v1 lines 79-117. HTTP helper with:
   - GitHub API headers (Accept: application/vnd.github+json, User-Agent)
   - Authorization Bearer header when GITHUB_TOKEN is set
   - Rate-limit retry on 403/429 with x-ratelimit-reset header parsing
   - Exponential backoff on other failures
   - MAX_RATE_LIMIT_WAIT_SECS = 60

2. **`pMap(items, fn, concurrency)`** -- Port from v1 lines 124-138. Bounded concurrency mapper.

3. **`parseSkillMdFrontmatter(content)`** -- Port from v1 lines 243-258. Parses YAML frontmatter
   from SKILL.md content, extracts `name` and `description` fields.

4. **`sleep(ms)`** -- Simple promise-based sleep (used by fetchJson).

5. **`getRepoTree(owner, repo, branch)`** -- Port from v1 lines 219-227. Fetches recursive tree
   via `https://api.github.com/repos/{owner}/{repo}/git/trees/{branch}?recursive=1`.

6. **`fetchSkillMdContent(owner, repo, branch, dirPath)`** -- Port from v1 lines 260-267. Fetches
   SKILL.md via Contents API, base64 decodes, calls parseSkillMdFrontmatter.

Add these constants at the top:

- `const GITHUB_TOKEN = process.env.GITHUB_TOKEN || ''`
- `const CONCURRENCY = 10`
- `const MAX_RATE_LIMIT_WAIT_SECS = 60`

Add `.env` loading at the top (port from v1 lines 18-29): read `.env` file from script's parent
directory, parse KEY=VALUE lines, set process.env if not already set.

Add a new async function `enrichWithGitHub(allSkills)` that:

a. Groups skills by repo: parse `source_url` to extract `owner/repo`, build a Map of repoFullName to skill[].
The v2 source_url is `https://github.com/{source}` where `source` = `owner/repo`.

b. Fetches repo metadata for each unique repo via `https://api.github.com/repos/{owner}/{repo}`
to get `default_branch`. Use pMap with CONCURRENCY.

c. Fetches repo tree for each repo via getRepoTree. Use pMap with CONCURRENCY.

d. For each skill, matches its `slug` to a directory in the tree. Search strategy:

- Look through all tree items of type 'tree' (directories)
- Find directories whose last path segment matches the skill's slug
- Prefer directories containing a SKILL.md (check tree blobs for `{dirPath}/SKILL.md`)
- If multiple matches, prefer paths under known scan bases: `skills/`, `skills/.curated/`,
  `skills/.experimental/`, `skills/.system/`, `.claude/skills/`

e. For matched skills, fetches SKILL.md content via fetchSkillMdContent. Use pMap with CONCURRENCY.

f. Updates each skill object in-place:

- `summary` = frontmatter `description` (if non-empty), otherwise keep empty
- `name` = frontmatter `name` (if non-empty), otherwise keep the slug
- `source_url` = `https://github.com/{owner}/{repo}/tree/{default_branch}/{dirPath}`
  (only when dirPath was found; if dirPath is null/root, keep repo URL)

g. Logs progress: "Enriching {N} skills across {M} repos...", per-repo tree fetch status,
final "Enriched {X}/{Y} skills with descriptions".

h. Returns gracefully if any step fails (no throw). Skills without matches keep their existing values.

In `main()`, call `enrichWithGitHub` after transform/dedup but before writing the JSON file:

```javascript
// After dedup, before write:
if (GITHUB_TOKEN) {
  const allSkills = [...skills, ...trending];
  await enrichWithGitHub(allSkills);
} else {
  console.warn(
    "GITHUB_TOKEN not set -- skipping SKILL.md enrichment (summaries will be empty)",
  );
}
```

IMPORTANT: Do NOT remove the Playwright scraping logic. The enrichment is additive -- Playwright
scrapes skills.sh for the skill list and install counts, then GitHub API enriches with descriptions.

IMPORTANT: The `source` field from skills.sh is `owner/repo` format. When constructing source_url
in transformSkill, continue using `https://github.com/${skill.source}` as the initial value --
the enrichment phase will upgrade it to include the full path.
</action>
