  <name>Task 2: Write fetch-featured-skills-v2.mjs scraper</name>
  <files>scripts/fetch-featured-skills-v2.mjs</files>
  <action>
Create `scripts/fetch-featured-skills-v2.mjs` that scrapes skills.sh leaderboard pages using Playwright and RSC payload extraction. Leave the original `scripts/fetch-featured-skills.mjs` untouched.

**Script structure:**

1. **Launch Playwright** (headless Chromium, library mode — NOT test runner):

   ```javascript
   import { chromium } from "playwright";
   ```

2. **Extract RSC payload function** — reusable for each page:
   - Navigate to URL with `page.goto(url)`
   - Wait for `domcontentloaded` (RSC data is in script tags in server-rendered HTML — no need for networkidle)
   - Get HTML via `page.content()`
   - Find `self.__next_f.push([1, "..."])` script tags containing `initialSkills`
   - Unescape the string content: `JSON.parse('"' + matchContent + '"')`
   - Find `{"initialSkills"` substring, parse brace-balanced JSON object
   - Return the `initialSkills` array (each entry: `{ source, skillId, name, installs }`)
   - Use the exact extraction algorithm from RESEARCH.md (verified working on all three pages)

3. **Scrape two pages:**
   - All Time: `https://skills.sh/` — produces the `skills` array (top 300 by all-time installs)
   - Hot: `https://skills.sh/hot` — produces the `trending` array (top 100 by recent momentum)

4. **Transform each skill entry** to match existing `featured-skills.json` schema:

   ```javascript
   {
     slug: skill.skillId,          // direct from RSC
     name: skill.name,             // direct from RSC (same as skillId on leaderboard)
     summary: '',                  // not available from leaderboard
     downloads: skill.installs,    // exact integer from RSC (e.g., 1242339)
     stars: 0,                     // not available from leaderboard
     category: 'general',          // not available from leaderboard
     tags: [],                     // not available from leaderboard
     source_url: `https://github.com/${skill.source}`,  // repo URL (per assumption A1)
     updated_at: timestamp,        // script run timestamp
   }
   ```

5. **Deduplicate** by slug (keep first occurrence, which is the higher-ranked one).

6. **Output JSON** to `featured-skills.json`:

   ```json
   {
     "updated_at": "ISO-8601 timestamp",
     "total": 300,
     "categories": ["general"],
     "skills": [ ...top 300 from All Time... ],
     "trending": [ ...top 100 from Hot... ]
   }
   ```

7. **Error handling:**
   - If RSC extraction fails for a page, log a clear error message (do not silently produce empty output)
   - If no skills extracted from any page, keep existing `featured-skills.json` (same fallback as v1)
   - Always close browser in a finally block

8. **Console output:** Log progress (page being scraped, skills found, dedup count, final output count).

**Important details:**

- Use `import { chromium } from 'playwright'` (ESM, library mode)
- No `.env` or GITHUB_TOKEN needed (this scrapes skills.sh, not GitHub API)
- Max 300 for `skills` array (All Time), max 100 for `trending` array (Hot)
- Do NOT use DOM traversal or virtual scroll expansion — RSC payload has all data
- The user's CONTEXT.md mentions expanding collapsed entries and clicking buttons, but research proved RSC extraction gets ALL skills without any interaction. Use RSC extraction as the primary (and only) approach, which fulfills the intent of "get full coverage" more reliably.
