Implementation must follow the exact same download strategy as `install_git_skill` (lines 110-258):

1. Call `parse_github_url(source_url)` to extract `clone_url`, `branch`, `subpath` -- same as line 117.
2. Resolve the explore-cache directory: `resolve_central_repo_path(app, store)?.join(".explore-cache")`. Create a subdirectory named by `repo_cache_key(source_url, None)` (reuse the existing private `repo_cache_key` function which does sha256 hashing -- encodes the full source_url so different subpaths within the same repo get separate cache entries). This falls under Claude's discretion per CONTEXT.md.
3. If the explore-cache skill dir already exists and is non-empty (has at least one file/dir besides `.git`), it's a cache hit -- return the path immediately. This gives instant re-view within the same session.
4. **Follow the same dual-path download strategy as `install_git_skill`:**
   - **GitHub API fast path (for subpath repos):** Call `parse_github_api_params(&parsed.clone_url, parsed.branch.as_deref(), parsed.subpath.as_deref())`. If it returns `Some((owner, repo, branch, subpath))`, attempt `download_github_directory()` to download ONLY the subpath directly into the explore-cache skill dir. On failure (except 404/cancellation), fall back to git clone. On 404, bail with an appropriate error (same as install_git_skill lines 184-191).
   - **Git clone path:** Call `clone_to_cache(app, store, &parsed.clone_url, parsed.branch.as_deref(), cancel)` to clone into the existing git cache (NOT the explore-cache -- reuse the installer's git cache for the repo clone). Then isolate the skill:
     - If there's a `subpath`: use `copy_dir_recursive(&repo_dir.join(subpath), &explore_skill_dir)` to copy the subpath into the explore-cache skill dir. Bail if subpath doesn't exist in repo.
     - If no subpath: check for multi-skill repos using `count_skills_in_repo(&repo_dir)`. If >= 2, bail with `MULTI_SKILLS|` error (same as install_git_skill line 248-250). If single-skill, use `copy_dir_recursive(&repo_dir, &explore_skill_dir)`.
5. Return the explore-cache skill dir path.

IMPORTANT implementation details:

- Use `GIT_CACHE_LOCK` for thread safety when checking/creating the explore-cache dir.
- For the GitHub API download path, get the github_token from store (same as install_git_skill lines 140-145) for rate limit avoidance.
- Do NOT register the skill in the database -- this is preview only.
- Do NOT rename based on SKILL.md name -- just return the raw path.
- Handle `CANCELLED|` propagation the same way as install_git_skill.
- Handle `RATE_LIMITED|` the same way as install_git_skill.

pub async fn clone_explore_skill(

    cancel: State<'_, Arc<CancelToken>>,
) -> Result<String, String> {

        Ok(path.to_string_lossy().to_string())
    })
    .await

}
