# Quick Task 260422-k78: Add skill detail viewer to Explore page - Context

**Gathered:** 2026-04-22
**Status:** Ready for planning

<domain>
## Task Boundary

Add the ability to view skill files (SKILL.md, code, etc.) directly from the Explore page, reusing the existing SkillDetailView component that currently only works for locally-installed skills in My Skills.

</domain>

<decisions>
## Implementation Decisions

### Data Fetching Strategy

- **MUST reuse the same installer layers** that handle non-standard repos: `parse_github_url()` for URL parsing + subpath extraction, GitHub API download for subpath repos, git clone fallback, and `copy_dir_recursive` for subpath isolation
- Cache destination: `~/.skillshub/.explore-cache/<owner>/<repo>/` (or subpath-qualified)
- Produces a real directory on disk that existing `list_skill_files` / `read_skill_file` commands can read from with zero changes
- New backend command: `clone_explore_skill` that returns the cache path pointing at the correct skill directory (not the repo root when a subpath is involved)
- Staleness check: `git fetch --depth 1` before reuse in same session
- Handles: nested skill paths, monorepos, GitHub shorthand URLs, tree/blob URL formats — same as real install

### Cache Lifecycle

- Cached clones persist for the duration of the app session
- On next app launch, startup cleanup wipes `.explore-cache/` directory
- Fits existing `cache_cleanup.rs` startup pattern
- Re-opening same skill in same session = instant (no re-clone)
- No user-facing cache management needed

### Viewer Integration

- Reuse `SkillDetailView` as-is — pass the explore-cache path as `centralPath`
- Construct a minimal ManagedSkill-like object with: central_path = cache path, name from explore card, source_ref = GitHub URL, source_type = 'github'
- Add optional "Install" button to SkillDetailView header when viewing an explore skill
- Back button returns to Explore page (not My Skills) when in explore-detail mode
- New view state `explore-detail` in App.tsx to distinguish from regular `detail` view

### Claude's Discretion

- Exact directory structure within `.explore-cache/`
- Whether to show a loading spinner during clone operation
- Error handling UX when clone fails (e.g., no network)

</decisions>

<specifics>
## Specific Ideas

- ExplorePage needs `onViewSkill` callback that carries the GitHub URL and skill name
- App.tsx needs new view state for `explore-detail` vs regular `detail`
- SkillDetailView could accept optional `isExplorePreview` prop to show Install button and adjust back navigation
- Featured skills and online search results both have `source_url` or equivalent to derive the clone URL from
