# Quick Task 260422-k78: Add Skill Detail Viewer to Explore Page Summary

Explore page View button that clones skills to a session-scoped cache and opens SkillDetailView with Install capability, reusing all installer download paths (GitHub API, git clone, subpath isolation, multi-skill detection).

## What Was Done

### Task 1: Backend (be626ba)

Added `clone_for_explore_preview()` to `installer.rs` that reuses the exact same download strategy as `install_git_skill`:

- Calls `parse_github_url()` for URL normalization and subpath extraction
- GitHub API fast path via `download_github_directory()` for subpath repos
- Git clone fallback via `clone_to_cache()` with subpath isolation via `copy_dir_recursive()`
- Multi-skill detection via `count_skills_in_repo()` with `MULTI_SKILLS|` error for ambiguous repos
- Thread safety via `GIT_CACHE_LOCK` mutex
- Session-scoped cache in `~/.skillshub/.explore-cache/<sha256-hash>/`
- Cache hit returns immediately if directory exists with content

Added `clone_explore_skill` Tauri command in `commands/mod.rs` with CancelToken support. Registered in `lib.rs` `generate_handler!`. Added explore-cache startup cleanup that wipes `.explore-cache/` on each app launch.

### Task 2: Frontend (1a5b779)

- Added `"explore-detail"` to `activeView` union type in App.tsx
- Added `handleOpenExploreDetail` callback: calls `clone_explore_skill`, constructs synthetic `ManagedSkill`, sets explore-detail view
- Added `handleBackToExplore` callback: returns to Explore page (not My Skills)
- Added `handleExploreInstallFromDetail` callback: triggers install then returns to Explore
- Added `onViewSkill` prop to `ExplorePage` with View button (Eye icon) on both featured and online search cards
- Added `isExplorePreview` and `onInstall` optional props to `SkillDetailView` with Install button in header
- Updated `Header` to highlight Explore tab when in `explore-detail` view
- Added `exploreView` i18n key (EN: "View", ZH: "View")
- Added CSS for `.explore-btn-view`, `.detail-header-actions`, `.detail-install-btn`
