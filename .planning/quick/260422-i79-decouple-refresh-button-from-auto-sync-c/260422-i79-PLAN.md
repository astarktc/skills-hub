<task type="auto">
  <name>Task 2: Replace handleRefresh with batch update-all + conditional sync</name>
  <files>src/App.tsx</files>
  <action>
Replace the existing `handleRefresh` callback (App.tsx ~lines 937-939) with the full batch implementation. The new function:

**Phase 1 -- Update all skills from source repos:**

1. Early return if `managedSkills.length === 0`
2. Set loading state: `setLoading(true)`, `setLoadingStartAt(Date.now())`, `setError(null)`
3. Initialize `collectedErrors: { title: string; message: string }[]`
4. Loop over `managedSkills` with index. For each skill:
   - `setActionMessage(t('actions.refreshStep', { index: i + 1, total: managedSkills.length, name: skill.name }))`
   - Call `await invokeTauri<UpdateResultDto>('update_managed_skill', { skillId: skill.id })`
   - On catch: push `{ title: t('errors.updateFailedTitle', { name: skill.name }), message: raw }` to `collectedErrors` and continue

**Phase 2 -- Conditional sync to tool directories (only if autoSyncEnabled):**
After the update loop, if `autoSyncEnabled`:

1. Re-fetch fresh skill data: `const freshSkills = await invokeTauri<ManagedSkill[]>('get_managed_skills')` -- this avoids stale `managedSkills` state (Pitfall 2 from RESEARCH.md)
2. Compute sync targets: `const syncTargetIds = uniqueToolIdsBySkillsDir(installedToolIds.filter((id) => isInstalled(id)))` -- reuse existing helpers
3. Only proceed with sync if `syncTargetIds.length > 0` and `freshSkills.length > 0`
4. Loop over `freshSkills` x `syncTargetIds`. For each pair:
   - `setActionMessage(t('actions.syncStep', { index: si + 1, total: freshSkills.length, name: skill.name, tool: toolLabel }))`
   - Call `await invokeTauri('sync_skill_to_tool', { sourcePath: skill.central_path, skillId: skill.id, tool: toolId, name: skill.name })`
   - On catch: skip `TOOL_NOT_INSTALLED|` and `TOOL_NOT_WRITABLE|` errors (continue), collect others with `t('errors.syncFailedTitle', { name, tool: toolLabel })` -- same pattern as `handleSyncAllManagedToTools`
   - Get `toolLabel` from `tools.find((t) => t.id === toolId)?.label ?? toolId`

**Wrap-up:**

- `setActionMessage(t('status.refreshCompleted'))`
- `setSuccessToastMessage(t('status.refreshCompleted'))`
- `setActionMessage(null)`
- `await loadManagedSkills()` -- final state refresh
- If `collectedErrors.length > 0`: `showActionErrors(collectedErrors)`
- In `finally` block: `setLoading(false)`, `setLoadingStartAt(null)`

**IMPORTANT -- do NOT call `handleSyncAllManagedToTools`** from within this function. It has its own loading state management that would conflict (Pitfall 1 from RESEARCH.md). Inline the sync logic instead.

**Dependency array:** `[autoSyncEnabled, installedToolIds, invokeTauri, isInstalled, loadManagedSkills, managedSkills, showActionErrors, t, tools, uniqueToolIdsBySkillsDir]`

The `onRefresh` prop type in FilterBar.tsx is `() => void`. An `async () => Promise<void>` is assignable to `() => void` in TypeScript, so no FilterBar changes needed. The current call site already uses `void` prefix pattern (`void handleRefresh()` equivalent via `onClick={onRefresh}`).
