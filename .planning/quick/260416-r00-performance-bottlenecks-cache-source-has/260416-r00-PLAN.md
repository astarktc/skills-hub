<objective>
Eliminate two independent performance bottlenecks in the project assignment matrix flow:

1. Backend: Stop recomputing source directory hashes on every assignment listing by using the already-cached `skills.content_hash` column.
2. Frontend: Replace O(A) linear scan per matrix cell with O(1) Map lookup via precomputed assignment map.

Purpose: The assignment matrix currently performs redundant filesystem I/O (hash_dir walks entire skill directories repeatedly) and redundant array scans (every row scans all assignments). These degrade noticeably as the number of skills and tools grows.

Output: Faster assignment listing and matrix rendering with zero schema changes.
</objective>
