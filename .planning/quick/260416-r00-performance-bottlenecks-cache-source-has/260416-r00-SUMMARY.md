# Quick Task 260416-r00: Performance Bottlenecks - Cache Source Hash and Precompute Assignment Map

Eliminated redundant filesystem I/O (hash_dir walks) and O(A) array scans in the project assignment matrix flow by using cached skill.content_hash for staleness checks and precomputing a Map for O(1) cell lookups.

## Changes Made

### Backend: Cached source hash lookups (installer.rs, project_sync.rs, skill_store.rs)

1. **installer.rs**: Removed `should_compute_content_hash()` debug/env guard function. `compute_content_hash()` now always computes the hash via `hash_dir()`. This ensures every installed or updated skill has a non-NULL `content_hash` in the database.

2. **project_sync.rs**: Rewrote `list_assignments_with_staleness()` with three optimizations:
   - **Skill record deduplication**: Pre-fetches skill records into a `HashMap<String, Option<SkillRecord>>` keyed by skill_id. Each unique skill gets exactly one DB query regardless of how many assignments reference it.
   - **Project record pre-fetch**: Fetches the project record once before the loop instead of per-iteration.
   - **Cached hash comparison**: Copy-mode staleness now reads `skill.content_hash` from the cached record instead of calling `hash_dir()`. For legacy records with NULL hash, falls back to `hash_dir()` and backfills the result via `update_skill_content_hash()`.

3. **skill_store.rs**: Added `update_skill_content_hash()` method for targeted hash backfill without requiring a full `upsert_skill()`.

### Frontend: Precomputed assignment Map (AssignmentMatrix.tsx)

1. Built `assignmentMap` via `useMemo` in the parent `AssignmentMatrix` component, keyed by `${skill_id}:${tool}`.
2. Replaced `MatrixRowProps.assignments: ProjectSkillAssignmentDto[]` with `assignmentMap: Map<string, ProjectSkillAssignmentDto>`.
3. Replaced O(A) `.find()` call per matrix cell with O(1) `.get()` on the Map.
4. Updated memo comparator to compare `assignmentMap` reference instead of `assignments`.

## Performance Impact

| Metric                                | Before                     | After                  |
| ------------------------------------- | -------------------------- | ---------------------- |
| `get_skill_by_id` calls per listing   | 1 per assignment           | 1 per unique skill     |
| `get_project_by_id` calls per listing | 1 per assignment           | 1 per call             |
| `hash_dir` calls per listing          | 1 per copy-mode assignment | 0 (when hashes cached) |
| MatrixRow cell lookup                 | O(A) linear scan           | O(1) Map.get           |
